Security posture (spec 10)
  • Bound to loopback only — non-loopback binds are rejected.
  • WebSocket Origin allow-listed; Host header validated on every route (anti DNS-rebinding).
  • All SQL parameterized; the chat bridge never receives raw SQL or a shell.
  • Database file created with 0600 permissions in a 0700 directory.
  • No authentication beyond loopback — a deliberate, documented v1 tradeoff.
